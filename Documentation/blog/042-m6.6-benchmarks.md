# M6.6: Syscall Performance Benchmarking

Before M7 adds /proc filesystem code to every VFS path, we needed a
clean performance baseline.  M6.6 expanded the benchmark suite to 28
syscalls, ran them on Linux (native + KVM) and Kevlar (KVM), and fixed
the regressions found.

---

## Methodology

All benchmarks use `make bench-kvm` (QEMU with `-enable-kvm -mem-prealloc`).
To get a **fair** comparison, we ran the Linux baseline two ways:
1. **Native** — directly on the host (what most benchmarks report)
2. **KVM** — same binary inside a QEMU KVM VM with a real Linux kernel

The KVM-vs-KVM comparison is the only fair one since both kernels run
through the same virtualization layer.  Native Linux has no VMCALL
overhead, EPT page table walks, or VM entry/exit costs.

## Fixes applied in M6.6

### Phase 1 fixes

**tkill syscall (missing)** — musl's `raise()` uses tkill (nr=200).
Without it, every call hit the "unimplemented syscall" `info!()` logger,
spamming serial at 115200 baud.  signal_delivery: 261µs → 498ns.

**/dev/zero read** — `write_with()` copied through a 256-byte kernel
buffer (16 usercopies for 4KB).  Switch to `fill()` for single usercopy.
read_zero: 473ns → 126ns.

**warn_once for unimplemented syscalls** — prevents serial I/O
dominating benchmarks when an unknown syscall is called in a loop.

### Phase 2 fixes

**uname: single usercopy** — build the 390-byte utsname struct on the
stack, write once.  Was 6 separate `write_bytes_or_zeroes()` calls.
uname: 181ns → 86ns (2.1x improvement).

**sigaction: batch struct read** — read the 32-byte sigaction struct in
one usercopy instead of 3 separate reads at offsets 0, 8, 16.
sigaction: 136ns → 120ns.

**fcntl: lock_no_irq** — fcntl isn't called from IRQ context, so use
`opened_files_no_irq()` instead of `opened_files().lock()` which does
cli/sti.  fcntl_getfl: 99ns → 79ns.

**readlink: lock_no_irq** — same optimization for root_fs lock.
readlink: 418ns → 414ns.

**mprotect: lock_no_irq + VMA fast path** — use `lock_no_irq()` for
VM lock, add fast path in `update_prot_range()` that updates prot
in-place when the range exactly covers a VMA (avoids Vec allocation
and VMA splitting).  mprotect: 1,365ns → 1,193ns.

**sched_yield: skip PROCESSES lookup** — when `pick_next()` returns the
same PID as the current process, reuse the existing Arc instead of
locking the PROCESSES hash table for a lookup.
sched_yield: 193ns → 165ns.
