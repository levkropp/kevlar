# M6.6: Syscall Performance Benchmarking

Before M7 adds /proc filesystem code to every VFS path, we needed a
clean performance baseline.  M6.6 expanded the benchmark suite to 28
syscalls, ran them on both Linux (native) and Kevlar (KVM), and fixed
the regressions found.

---

## Fixes applied

### signal_delivery: 261,104ns → 493ns (530x improvement)

Root cause: `raise()` calls musl's `tkill()` syscall (nr=200), which
was unimplemented.  The "unimplemented syscall" handler printed a log
line to the serial console for EVERY call.  At 115200 baud, each 45-char
message takes ~400µs.  With 200K iterations, that's 80 seconds of serial
I/O.

Fix: implemented `sys_tkill` (delegates to `sys_tgkill`).  Also changed
the unimplemented syscall log from `info!()` to `warn_once!()` to
prevent serial spam for any future missing syscalls.

After the fix, signal delivery is 2.4x FASTER than Linux — Kevlar's
signal path is lean (no sigaltstack, no siginfo, minimal context save).

### read_zero: 473ns → 125ns (3.8x improvement)

Root cause: `UserBufWriter::write_with()` copies through a 256-byte
kernel buffer — for a 4KB `/dev/zero` read, that's 16 usercopy calls.

Fix: use `UserBufWriter::fill(0, len)` which calls `UserVAddr::fill()`
directly, filling userspace memory in a single usercopy.  Now 1.5x
FASTER than Linux.

## Remaining regressions (acceptable)

| Benchmark | Ratio | Reason |
|-----------|-------|--------|
| mmap_fault | 2.01x | EPT page table walks (KVM architectural cost) |
| mprotect | 1.32x | VMA range update + PTE walk + TLB flush |
| readlink | 1.12x | Extra VFS path resolution overhead |

`mmap_fault` is an unavoidable KVM cost — every demand-paged fault
requires EPT page table walks in addition to guest page table walks.
Linux's baseline was measured on bare metal, not under KVM.

## Highlights

- **brk: 380x faster** — Kevlar's brk just adjusts a VMA length field.
  Linux's brk does page table manipulation, VMA splitting, and
  accounting.
- **mmap_munmap: 5.5x faster** — Kevlar's mmap is a VMA append, munmap
  is a VMA remove.  No page table manipulation until first fault.
- **getcwd: 2.0x faster** — Kevlar's cwd is a simple string copy from
  the process struct.
